## 🚀 立即启用自动记录

### 方案 1: Git 活动追踪（最简单）

这是最推荐的自动记录方式，无需额外配置。

#### 工作原理
```
您正常写代码 → Git 提交 → 自动捕获 commit → 分析并归档 → 生成日报
```

#### 自动提取的信息
从每个 Git commit 中自动提取：
- ✅ **项目名称** - 从仓库路径识别
- ✅ **任务类型** - 从 commit message 识别（feat/fix/docs 等）
- ✅ **工作内容** - commit 描述
- ✅ **技术栈** - 从修改的文件识别
- ✅ **工作量** - 提交数量和代码量
- ✅ **时间分布** - 提交时间统计

#### 示例
```bash
# 您正常的 Git 操作
cd D:\work\code\alliance-admin-web
git add .
git commit -m "feat: 重构权限管理模块，添加 RBAC 角色控制"
git push

# 系统自动记录:
✅ [FEATURE] [HIGH] alliance-admin-web
   - 工作内容：重构权限管理模块，添加 RBAC 角色控制
   - 技术栈：Vue.js, Vuex, Permission Control
   - 时间：2026-04-02 14:30
   - 代码量：+328, -156
```

#### 启用方法
只需告诉我：
```
"开始监控我的 Git 活动"
```

我会自动扫描您所有项目的 Git 记录并归档。

## 💡 常见问题

### Q: 自动监控会泄露隐私吗？
**A**: 
- ❌ 不会记录具体内容（代码、文档等）
- ✅ 只记录元数据（文件名、时间、项目名）
- ✅ 所有数据本地存储
- ✅ 可随时关闭监控

### Q: 会影响电脑性能吗？
**A**: 
- ✅ Git 监控：几乎零开销（只在提交时触发）
- ✅ 文件监控：<1% CPU，极低内存占用
- ✅ 都是后台静默运行，无感知

### Q: 如果忘记 Git 提交怎么办？
**A**: 
- 文件监控会补充记录未提交的工作
- 也可以手动补充："今天上午做了 XXX"
- 系统会自动整合所有数据源

### Q: 可以临时关闭监控吗？
**A**: 
- ✅ 随时可以说："暂停监控"
- ✅ 也可以指定时间段："下午 2-4 点暂停"
- ✅ 完全由您控制
